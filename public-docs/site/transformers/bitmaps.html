    Bitmaps¶

    The bitmap effect displays sprite images on LED matrices. Bitmaps can be defined inline as string arrays, loaded from GIF files, or loaded from JSON sprite files extracted from ROM data.

    Inline Bitmaps¶

    Bitmaps can be defined directly as string arrays. Each character is a hex palette index (0–F), and . represents a transparent pixel:

    broadcast({
  effect: 'bitmap',
  props: {
    reset: false,
    centerX: 0,
    centerY: 0,
    duration: 1500,
    fadeIn: 300,
    fadeOut: 300,
    images: [[
      '.......A........',
      '......AAA.......',
      '....BBBBBAAAA...',
      '...BBBBBBBAA....',
      '..B7B77BBBB.....',
      '..70B077BBBAAA..',
      '..70B077BBBBA...',
      '.B70B077AABB....',
      '.A70B077AAABAA..',
      '.BB7B77BA0BBA...',
      '..0070000BEB....',
      '..BBBBBBBEEEB...',
      '...7777BEEEEB...',
      '..777777BEEBBA..',
      'EEE777EEBBBBBBA.',
      '.EE77EEEEEBBBBBB',
    ]],
  },
});

    This is useful for small, hand-crafted sprites or when you want to embed bitmap data directly in a transformer without managing separate files.

    When no palette is provided, the bitmap effect uses the PICO-8 palette as the default. The hex characters in your image data map directly to these 16 colors:

    0 Black 1 Dark Blue 2 Purple 3 Dark Green 4 Brown 5 Dark Gray 6 Light Gray 7 White 8 Red 9 Orange A Yellow B Green C Blue D Lavender E Pink F Peach

    Loading JSON Sprites¶

    The loadSprite function loads JSON sprite files and converts them to the bitmap effect format. JSON sprites are typically generated by the sprite extraction system, which reads sprite graphics directly from MAME ROM data at runtime.

    const sprite = await loadSprite('bitmaps/pac-right.json');

    The function is available on the transformer context object:

    export async function transform({ subject, property, payload }, { broadcast, loadSprite }) {
  const sprite = await loadSprite('bitmaps/my-sprite.json');
  // ...
}

    Parameters¶

    Parameter Type Description
    path string Path to the JSON sprite file, relative to the transformers/ folder in your config directory

    Return Value¶

    Returns the same result format as loadGif:

    Property Type Description
    images string[][] Array of frames. Each frame is an array of row strings using hex characters (0-F) for palette indices and spaces for transparent pixels.
    palette string[] Array of up to 16 hex color strings. Optional — when absent, the default PICO-8 palette is used.
    width number Width in pixels (derived from the widest row across all frames).
    height number Height in pixels (derived from the number of rows in the first frame).
    frameCount number Number of frames (derived from the length of the images array).

    Note

    The JSON files themselves only contain images and an optional palette. The width, height, and frameCount properties are computed automatically when the file is loaded.

    Using with Bitmap Effect¶

    const sprite = await loadSprite('bitmaps/pac-right.json');

broadcast({
  effect: 'bitmap',
  props: {
    images: sprite.images,
    palette: sprite.palette,
    centerX: 50,
    centerY: 50,
    duration: 2000,
  },
});

    When the sprite has no custom palette (common for ROM-extracted sprites that use color_map), the driver renders using the default PICO-8 palette.

    Animation Support¶

    Multi-frame JSON sprites work the same way as animated GIFs. Set frameRate to control playback speed:

    const ghost = await loadSprite('bitmaps/ghost-scared.json');

broadcast({
  effect: 'bitmap',
  props: {
    images: ghost.images,
    palette: ghost.palette,
    frameRate: 4,
    centerX: 50,
    centerY: 50,
    duration: 3000,
  },
});

    JSON File Format¶

    JSON sprite files contain an images array and an optional palette:

    {
  "images": [
    [
      " AA ",
      "AAAA",
      "A  A"
    ]
  ],
  "palette": ["#FF004D", "#FFA300", "#FFEC27", "#00E436"]
}

    For multi-frame sprites, the images array contains multiple frames:

    {
  "images": [
    ["AA", "AA"],
    [" A", "A "]
  ]
}

    Loading GIF Files¶

    The loadGif function loads animated GIF files and converts them to the bitmap effect format.

    const sprite = await loadGif('bitmaps/cherry.gif');

    The function is available on the transformer context object:

    export async function transform({ subject, property, payload }, { broadcast, loadGif }) {
  const sprite = await loadGif('bitmaps/my-sprite.gif');
  // ...
}

    Parameters¶

    Parameter Type Description
    path string Path to the GIF file, relative to the transformers/ folder in your config directory

    Return Value¶

    Returns the same result format as loadSprite, plus a frameRate for animated GIFs:

    Property Type Description
    images string[][] Array of frames. Each frame is an array of row strings using hex characters (0-F) for palette indices.
    palette string[] Array of up to 16 hex color strings extracted from the GIF's color table.
    width number Width of the GIF in pixels.
    height number Height of the GIF in pixels.
    frameCount number Number of frames in the GIF.
    frameRate number Frames per second (only present for animated GIFs with more than 1 frame).

    Using with Bitmap Effect¶

    The result from loadGif provides everything needed for the bitmap effect:

    const sprite = await loadGif('bitmaps/cherry.gif');

broadcast({
  effect: 'bitmap',
  props: {
    images: sprite.images,
    palette: sprite.palette,
    centerX: 50,
    centerY: 50,
    duration: 2000,
  },
});

    Animation Support¶

    For animated GIFs, the frameRate property indicates the playback speed:

    const animatedSprite = await loadGif('bitmaps/explosion.gif');

broadcast({
  effect: 'bitmap',
  props: {
    images: animatedSprite.images,
    palette: animatedSprite.palette,
    frameRate: animatedSprite.frameRate,  // Use GIF's timing
    centerX: 50,
    centerY: 50,
    duration: 2000,
  },
});

    Override frameRate in props to change playback speed:

    broadcast({
  effect: 'bitmap',
  props: {
    images: animatedSprite.images,
    palette: animatedSprite.palette,
    frameRate: 12,  // Force 12 FPS regardless of GIF timing
    // ...
  },
});

    Color Palette Limitations¶

    • GIFs are converted to a 16-color palette
    • Colors beyond the first 16 in the GIF's color table are mapped using modulo
    • Transparent pixels are preserved (rendered as spaces in the bitmap data)

    Caching and Initialization¶

    Sprite files should be loaded once and cached for reuse. The recommended pattern is to load sprites when the game initializes using the init event, then cache them in module-level variables:

    let pacRight, ghostScared;

export async function transform({ subject, property, payload }, { broadcast, loadSprite }) {
  // Load sprites on game init (or on first event if not yet loaded)
  if (subject === 'init' || !pacRight) {
    [pacRight, ghostScared] = await Promise.all([
      loadSprite('bitmaps/pac-right.json'),
      loadSprite('bitmaps/ghost-scared.json'),
    ]);
    return true;
  }

  if (subject === 'player' && property === 'eat-dot') {
    return broadcast({
      effect: 'bitmap',
      props: {
        images: pacRight.images,
        centerX: 50,
        centerY: 50,
        duration: 500,
      },
    });
  }

  return false;
}

    The init event is emitted automatically when a game starts. Loading sprites in the init handler ensures they're ready before gameplay events arrive, and reloads them if the sprite files change during development.

    For sprites loaded on demand (like bonus items that vary), cache after first load:

    const BONUS_ITEMS = {
  cherry: { score: 100, file: 'pac-bonus-1-cherry.json' },
  strawberry: { score: 300, file: 'pac-bonus-2-strawberry.json' },
};

// Inside transform():
if (subject === 'player' && property === 'eat') {
  const bonusItem = BONUS_ITEMS[payload];

  if (bonusItem) {
    if (!bonusItem.sprite) {
      bonusItem.sprite = await loadSprite(`bitmaps/${bonusItem.file}`);
    }

    broadcast({
      effect: 'bitmap',
      props: {
        images: bonusItem.sprite.images,
        palette: bonusItem.sprite.palette,
        centerX: 50,
        centerY: 50,
        duration: 1000,
      },
    });
  }
}

    File Location¶

    Place bitmap files in subdirectories under transformers/ in your config directory:

    transformers/
├── bitmaps/
│   ├── pac-right.json        ← Generated by sprite extraction
│   ├── ghost-scared.json     ← Generated by sprite extraction
│   ├── explosion.gif         ← Hand-crafted GIF
│   └── custom-sprite.json    ← Hand-crafted JSON
└── games/
    └── pacman.js

    JSON sprite files in bitmaps/ are typically generated by the sprite extraction system when MAME loads a game. You can also create JSON files by hand following the file format above.

    Error Handling¶

    Handle loading errors gracefully to prevent transformer failures:

    try {
  const sprite = await loadSprite('bitmaps/bonus.json');
  // Use sprite...
} catch (err) {
  console.error('Failed to load sprite:', err);
  // Fallback behavior or skip the effect
}

    Common errors:

    • File not found at the specified path
    • Invalid JSON or missing images array
    • For GIFs: invalid or corrupted file, no frames, no color table

    Movement and Easing¶

    Bitmaps can animate from a start position to an end position over the duration. Set endX and/or endY to define the destination:

    broadcast({
  effect: 'bitmap',
  props: {
    images: sprite.images,
    palette: sprite.palette,
    centerX: 0,
    centerY: 50,
    endX: 100,
    endY: 50,
    duration: 2000,
    easing: 'cubicInOut',
  },
});
    Property Type Default Description
    endX number | 'random' — End X position (0–100 or 'random'). If omitted, the bitmap stays at centerX.
    endY number | 'random' — End Y position (0–100 or 'random'). If omitted, the bitmap stays at centerY.
    easing string 'quadraticInOut' Easing curve for the movement. Only applies when endX or endY is set.
